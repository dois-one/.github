# Contributing to DOIS

DOIS is primarily a personal content platform by Thiago and Andreos. However, we welcome contributions in specific areas.

## How to Contribute

### Bug Reports
If you find a bug on the website, please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

### Feature Suggestions
Have an idea to improve the reading experience? Open an issue tagged `enhancement` with:
- Problem you're trying to solve
- Proposed solution
- Any alternatives considered

### Code Contributions
For code contributions:
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests and ensure build passes
5. Commit with clear messages
6. Open a Pull Request

## What We're Looking For

- Accessibility improvements
- Performance optimizations
- Bug fixes
- Documentation improvements

## What We're NOT Looking For

- Content submissions (we write our own articles)
- Major architectural changes without prior discussion
- Features that add complexity without clear value

## Code Style

- TypeScript for all code
- Tailwind CSS for styling
- Follow existing patterns in the codebase
- Keep changes minimal and focused

## Questions?

Open an issue or reach out through the website.
